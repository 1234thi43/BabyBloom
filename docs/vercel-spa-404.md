# Исправление 404 на Vercel для React Router (SPA)

## Проблема

После деплоя на Vercel ссылка на `/portfolio` (и другие маршруты) даёт **404 Not Found**, хотя локально всё работает.

## Причина

- Проект — **SPA** (Vite + React Router). Реальные маршруты есть только в JS в браузере.
- После `vite build` в `dist` лежит в основном **`index.html`** и ассеты, отдельной страницы `/portfolio` на диске **нет**.
- **Vite dev server** для любого пути отдаёт `index.html` — поэтому локально ок.
- **Vercel** при запросе `/portfolio` ищет статический файл по этому пути — не находит — **404**.

Пример проблемной ссылки: `GalleryPreview.tsx` — `<motion.a href="/portfolio">` делает **полную навигацию** по URL, сервер должен отдать `index.html` для этого пути.

## Решение (обязательно для продакшена)

В **корне репозитория** (рядом с `package.json`) создан файл **`vercel.json`**:

```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

После пуша Vercel пересоберёт проект; прямые ссылки и F5 на вложенных маршрутах должны работать.

## Дополнительно (код)

- Заменить `motion.a` + `href="/portfolio"` на **`Link`** из `react-router-dom` для клиентской навигации — удобнее, но **не заменяет** `vercel.json`: без rewrites закладка или F5 на `/portfolio` снова даст 404.

## Проверка

После деплоя открыть в браузере напрямую: `https://<ваш-домен>.vercel.app/portfolio` — должна открыться страница портфолио, а не 404.
